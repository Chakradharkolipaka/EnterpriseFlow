🎯 DEPLOYMENT STRATEGY
────────────────────────────────────────────────────────────────
Both frontend and backend will be deployed to Vercel:

Backend:  https://your-backend.vercel.app
Frontend: https://your-frontend.vercel.app
Database: MongoDB Atlas (Free M0 cluster)

⏱️  ESTIMATED TIME: 20-30 minutes total

📋 QUICK STEPS
────────────────────────────────────────────────────────────────
1. Setup MongoDB Atlas (5 min)
   → Create free cluster
   → Configure network access
   → Get connection string

2. Push to GitHub (2 min)
   → git add .
   → git commit -m "Ready for deployment"
   → git push origin main

3. Deploy Backend (5 min)
   → Import repo to Vercel
   → Set root: backend
   → Add environment variables
   → Deploy

4. Seed Database (3 min)
   → Run: npm run seed
   → Creates 4 users + sample data

5. Deploy Frontend (5 min)
   → Import repo to Vercel again
   → Set root: frontend
   → Set VITE_API_URL
   → Deploy

6. Update CORS (2 min)
   → Update backend CORS_ORIGIN
   → Redeploy backend

7. Test Everything (5 min)
   → Login as all 4 roles
   → Create customer
   → Create product
   → Create challan

🌐 ENVIRONMENT VARIABLES
────────────────────────────────────────────────────────────────
Backend (.env on Vercel):

  MONGODB_URI=mongodb+srv://...
  JWT_SECRET=your_secret_key_here
  JWT_EXPIRES_IN=7d
  CORS_ORIGIN=https://your-frontend.vercel.app

Frontend (.env on Vercel):
  VITE_API_URL=https://your-backend.vercel.app/api

💡 TIPS
────────────────────────────────────────────────────────────────
• Deploy backend BEFORE frontend (frontend needs backend URL)
• Test backend health endpoint after deploy
• Seed database before testing frontend
• CORS_ORIGIN must match frontend URL exactly
• Use strong JWT secret in production
• Monitor Vercel logs for any errors
