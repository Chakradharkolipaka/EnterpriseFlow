WHY THIS FIXES THE "_id" ERROR
═══════════════════════════════════════════════════════════════════

Root Cause:
- Serverless functions are stateless
- Each invocation might have no DB connection
- Mongoose tries to buffer commands → fails
- Models fail to initialize properly → _id error

Solution:
- Explicit connectDB() call before operations
- Connection caching (reuse between invocations)
- bufferCommands: false (fail fast if no connection)
- maxPoolSize: 1 (optimal for serverless)
